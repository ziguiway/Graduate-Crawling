## 方法一：相关性分析（最简单）

### 原理

算每个特征和目标值之间的**相关系数**，取绝对值越大，说明这个特征和目标越"步调一致"。

**公式**（Pearson 相关系数）：
$$
r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2}\sqrt{\sum(y_i - \bar{y})^2}}
$$

- r = 1：完全正相关（特征涨，目标也涨）
- r = -1：完全负相关（特征涨，目标跌）
- r = 0：没线性关系

### 代码（完整可运行）

```python
# 方法一：相关性分析
# 目的：看每个特征和目标"步调一致"的程度
import pandas as pd

df = pd.read_csv('covid.train.csv')
target = 'tested_positive.2'

# ---- 1. 算相关性 ----
feature_cols = [c for c in df.columns if c not in ['id', target]]
corrs = df[feature_cols].corr()[target].sort_values(key=abs, ascending=False)

# ---- 2. 输出列名 + 下标（去掉 id 后从 0 开始） ----
col_to_idx = {col: i - 1 for i, col in enumerate(df.columns) if col != 'id'}

print('===== 和目标相关性最高的特征 =====')
print(f'{\"列名\":25s}  {\"下标\":>5s}  {\"相关性\":>8s}')
print('-' * 45)
for col in corrs.head(15).index:
    idx = col_to_idx[col]
    r = corrs[col]
    print(f'{col:25s}  {idx:5d}  {r:8.4f}')

print()
print('===== 和目标相关性最低的特征 =====')
print(f'{\"列名\":25s}  {\"下标\":>5s}  {\"相关性\":>8s}')
print('-' * 45)
for col in corrs.tail(10).index:
    idx = col_to_idx[col]
    r = corrs[col]
    print(f'{col:25s}  {idx:5d}  {r:8.4f}')
```

### 深挖：相关性到底告诉我们什么？

**1. Pearson 相关系数只能捕捉线性关系**

如果特征和目标的关系是 y = x²（抛物线），Pearson r ≈ 0，但特征其实完全预测了目标。所以做相关性分析前，先画散点图矩阵看关系形态。

**2. 高相关 ≠ 因果**

`tested_positive.1` 和 `tested_positive.2` 相关 0.99，不是说昨天的确诊"导致"了今天的——而是疫情本来就是连续过程，今天是昨天的延续。这叫**自相关（autocorrelation）**，时间序列的固有属性。

**3. 相关性会受异常值严重干扰**

一个离谱的点可以把 r 从 0.8 拉到 0.2。务必先检查数据分布。

**4. Spearman 秩相关（进阶）**

不要求线性关系，只要求**单调关系**：
$$
r_s = 1 - \frac{6\sum d_i^2}{n(n^2 - 1)}
$$

其中 d_i 是第 i 个样本在两个变量上的**排名差**。适用于数据有明显非线性趋势的情况。

## 方法二：线性模型权重（看模型学到了啥）

### 原理

训练一个线性回归（带 L2 正则化的 Ridge），然后看每个特征的**权重绝对值**。权重越大，模型越依赖这个特征。

和相关性分析的区别：相关性是单变量分析（只看特征和目标的关系），权重是**多变量分析**（看所有特征一起时各个特征有多大贡献）。可能有特征的权重和相关性符号不同——那就是和其他特征相互作用的结果。

```
worried_finances          0.29
hh_cmnty_cli.1            0.27
wearing_mask.1            0.26
work_outside_home.2       0.26
```

### 深挖：这个方法的陷阱

**1. 权重不稳定：共线性问题**

假设两个特征 x₁ 和 x₂ 高度相关（如 cli 和 ili，相关性 0.97+）。线性回归：
$$
y = w_1 x_1 + w_2 x_2 + b
$$

如果 x₁ ≈ x₂，那 w₁ = 1, w₂ = 0 和 w₁ = 0, w₂ = 1 的预测效果几乎一样。权重分配是**任意的**，取决于初始化和随机噪声。

这就是**多重共线性（multicollinearity）**。症状：整体预测很好，但单个特征的权重不可靠。

**检测方法——VIF（方差膨胀因子）：**
```python
from statsmodels.stats.outliers_influence import variance_inflation_factor

vif = pd.DataFrame()
vif['feature'] = X.columns
vif['VIF'] = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]
```
VIF > 10 → 严重共线性，该特征的权重不可信。

**2. 为什么用 Ridge 不用普通线性回归？**

Ridge 加了 L2 正则项：
$$
\min \sum(y - \hat{y})^2 + \lambda\sum w_j^2
$$

效果：当特征高度相关时，Ridge 把权重**均匀摊**给它们，而不是让一个极大一个极小。这样权重更稳定、更可解释。

**3. 尺度问题**

特征的尺度直接影响权重大小。`spent_time` 的范围是 0~100，`wearing_mask` 是 0~1。即使两个特征同样重要，`wearing_mask` 的权重会比 `spent_time` 大 100 倍。所以**一定要先标准化再解读权重**。

### 代码（完整可运行）

```python
# 方法二：线性模型权重
# 目的：训练一个线性模型，看每个特征的权重（越大越重要）
import pandas as pd
import numpy as np
from sklearn.linear_model import Ridge
from sklearn.preprocessing import StandardScaler

df = pd.read_csv('covid.train.csv')
target = 'tested_positive.2'

# 列名 → 下标（去掉 id 后从 0 开始）
col_to_idx = {col: i - 1 for i, col in enumerate(df.columns) if col != 'id'}

# 准备数据
X = df.drop(columns=['id', target])
y = df[target]

# 标准化（不标准化的话权重不可比）
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 训练 Ridge 回归（加 L2 正则让权重更稳定）
model = Ridge(alpha=1.0)
model.fit(X_scaled, y)

# 按权重绝对值排序，输出列名 + 下标
importance = pd.DataFrame({
    '特征': X.columns,
    '下标': [col_to_idx[c] for c in X.columns],
    '权重': model.coef_,
    '|权重|': np.abs(model.coef_)
}).sort_values('|权重|', ascending=False)

print('===== Ridge 回归：最重要的 15 个特征 =====')
print(importance.head(15).to_string(index=False))
print()
print('===== Ridge 回归：最不重要的 10 个特征 =====')
print(importance.tail(10).to_string(index=False))
```

## 方法三：Permutation Importance（置换重要性，最可靠）

### 原理

1. 先训练好模型，算 baseline loss
2. **把某一个特征的值打乱**（破坏它和目标的关联）
3. 再算 loss，看上升多少
4. 上升越多 → 这个特征越重要

**打个比方：** 你想知道方向盘对开车重不重要，就把方向盘拆了开一段路，看还能不能正常开。

**正式定义：**
$$
PI(x_j) = \mathbb{E}[L(Y, f(X^{(\text{perm }j)}))] - \mathbb{E}[L(Y, f(X))]
$$

其中 X^{(\text{perm }j)} 是把第 j 列打乱后的数据。PI 是**期望 loss 的增加量**。

### 为什么它最好？

相关性和权重都只看**线性关系**。Permutation Importance 不管你的模型是线性、神经网络还是随机森林，**统统都能用**，直接反映"没这个特征模型会差多少"。

### 深挖：这个方法的陷阱

**1. 特征高度相关时，PI 会低估重要性**

思考：如果 x₁ 和 x₂ 相关系数 0.98，你把 x₁ 打乱了——x₂ 还有几乎一样的信息。loss 几乎不变。但你能说 x₁ 不重要吗？

**这和"共线性让权重不稳定"是同一个问题的两面：**
- 线性模型权重：高共线性 → 权重分配不可靠
- Permutation Importance：高共线性 → 认为两个特征都不重要（但其实至少有一个重要）

**怎么解决？** 分组打乱（group permutation）：把相关系数高的特征放在一组，一次性全部打乱。

**2. PI 的随机性**

打乱是随机的，每次跑的结果会有波动。建议跑多次取平均：
```python
results = []
for _ in range(5):
    imp = permutation_importance(model, X_val, y_val)
    results.append(imp)
importance = np.mean(results, axis=0)
```

### 代码（完整可运行）

```python
# 方法三：Permutation Importance（置换重要性）
# 目的：把每个特征打乱，看在验证集上 loss 涨多少
# 涨越多 → 特征越重要
import numpy as np
import pandas as pd
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset

# ---------- 1. 数据集 ----------
class COVID19Dataset(Dataset):
    def __init__(self, file_path, mode):
        data = pd.read_csv(file_path).values[:, 1:].astype(float)
        target = data[:, -1]
        data = data[:, :]  # 用全部 93 个特征
        
        if mode == 'train':
            idx = [i for i in range(len(data)) if i % 10 != 0]
            self.data = torch.FloatTensor(data[idx])
            self.target = torch.FloatTensor(target[idx])
        else:  # dev
            idx = [i for i in range(len(data)) if i % 10 == 0]
            self.data = torch.FloatTensor(data[idx])
            self.target = torch.FloatTensor(target[idx])
        
        # 归一化
        mean = self.data.mean(dim=0, keepdim=True)
        std = self.data.std(dim=0, keepdim=True)
        std[std == 0] = 1
        self.data = (self.data - mean) / std
        self.dim = self.data.shape[1]
    
    def __getitem__(self, idx):
        return self.data[idx], self.target[idx]
    def __len__(self):
        return len(self.data)

# ---------- 2. 简单的神经网络 ----------
class SimpleNet(nn.Module):
    def __init__(self, input_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 64), nn.ReLU(),
            nn.Linear(64, 64), nn.ReLU(),
            nn.Linear(64, 1)
        )
    def forward(self, x):
        return self.net(x).squeeze(1)

# ---------- 3. 训练 ----------
device = 'cuda' if torch.cuda.is_available() else 'cpu'
print(f'设备: {device}')

tr_set = DataLoader(COVID19Dataset('covid.train.csv', 'train'), batch_size=128, shuffle=True)
dv_set = DataLoader(COVID19Dataset('covid.train.csv', 'dev'), batch_size=128)

model = SimpleNet(tr_set.dataset.dim).to(device)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.MSELoss()

# 训练 100 个 epoch
best_loss = float('inf')
for epoch in range(100):
    model.train()
    for x, y in tr_set:
        x, y = x.to(device), y.to(device)
        optimizer.zero_grad()
        loss = criterion(model(x), y)
        loss.backward()
        optimizer.step()
    
    # 验证
    model.eval()
    total = 0
    with torch.no_grad():
        for x, y in dv_set:
            x, y = x.to(device), y.to(device)
            total += criterion(model(x), y).item() * len(x)
    dev_loss = total / len(dv_set.dataset)
    
    if dev_loss < best_loss:
        best_loss = dev_loss
    if epoch % 20 == 0:
        print(f'epoch {epoch:3d}, dev loss: {dev_loss:.4f}')

print(f'训练完成，最佳 dev loss: {best_loss:.4f}')
print()

# ---------- 4. Permutation Importance ----------
# 对每个特征，打乱它，重新算 loss，看涨了多少
all_cols = list(pd.read_csv('covid.train.csv').columns[1:])
dv_dataset = dv_set.dataset  # 直接操作 dataset 的数据
baseline = best_loss

print('===== Permutation Importance =====')
for feat_idx in range(dv_dataset.dim):
    # 保存原始数据
    orig = dv_dataset.data[:, feat_idx].clone()
    
    # 打乱这个特征
    dv_dataset.data[:, feat_idx] = orig[torch.randperm(len(dv_dataset))]
    
    # 算打乱后的 loss
    model.eval()
    total = 0
    with torch.no_grad():
        for x, y in dv_set:
            x, y = x.to(device), y.to(device)
            total += criterion(model(x), y).item() * len(x)
    perm_loss = total / len(dv_dataset)
    
    # 恢复
    dv_dataset.data[:, feat_idx] = orig
    
    # 打印重要特征（loss 上涨超过 0.01 的）
    increase = perm_loss - baseline
    if increase > 0.01:
        bar = '█' * int(increase * 20)
        print(f'{all_cols[feat_idx]:25s}  loss ↑{increase:.4f}  {bar}')
```

## 进阶方法：SHAP（当前学术界最推荐的方法）

### 核心思想

Permutation Importance 只给你一个**全局**数字——"这个特征平均重要多少"。SHAP 告诉你**每个样本**上每个特征的贡献——"这个样本上，为什么模型预测高了/低了"。SHAP 的理论基础是博弈论中的 Shapley Value。

### Shapley Value 的直觉

假设四个队友合作完成项目，总收入 100 万。怎么公平分钱？Shapley 的方法：看每个人在所有可能的"加入顺序"中带来的**边际贡献**的平均值。

换成特征分析：预测值是模型所有特征合作的产出。SHAP 计算每个特征在所有特征子集组合中的边际贡献的平均值。有 93 个特征 → 2⁹³ 种组合 → 靠采样近似。

### SHAP 能干什么

**1. SHAP Summary Plot——全局重要性 + 影响方向**

```python
# feature_analysis_04_shap.py
# SHAP 分析 - 完整可运行脚本
import shap
import pandas as pd
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset
import matplotlib.pyplot as plt

# ========== 1. 复用模型和数据 ==========
# （这里复用方法三中的 COVID19Dataset 和 NeuralNet）
# 简化起见，用你已经训练好的模型

class COVID19Dataset(Dataset):
    """和上面方法三一样的数据集定义，省略... 实际使用时直接复制"""

class NeuralNet(nn.Module):
    """和上面方法三一样的模型定义，省略..."""

# ========== 2. 加载模型 ==========
device = 'cuda' if torch.cuda.is_available() else 'cpu'
dv_dataset = COVID19Dataset('covid.train.csv', 'dev', use_all_features=True)
model = NeuralNet(dv_dataset.dim).to(device)
model.load_state_dict(torch.load('models/model.pth', map_location=device, weights_only=True))
model.eval()

# ========== 3. 准备 SHAP 需要的数据 ==========
dv_loader = DataLoader(dv_dataset, batch_size=128)

all_x, all_y = [], []
for x, y in dv_loader:
    all_x.append(x)
    all_y.append(y)
X_val = torch.cat(all_x, dim=0).numpy()
y_val = torch.cat(all_y, dim=0).numpy()

# 特征名
all_cols = list(pd.read_csv('covid.train.csv').columns[1:])
feats = list(range(93))  # 如果用全部特征

# ========== 4. 创建 SHAP Explainer ==========
# 方法 A: DeepExplainer（适用于 PyTorch 模型，需要训练数据做背景）
background = X_val[:100]  # 取 100 个样本做背景
explainer = shap.DeepExplainer(model, torch.FloatTensor(background).to(device))

# 方法 B: KernelExplainer（模型无关，更通用但更慢）
# explainer = shap.KernelExplainer(
#     lambda x: model(torch.FloatTensor(x).to(device)).detach().cpu().numpy(),
#     background
# )

# ========== 5. 计算 SHAP 值 ==========
print('计算 SHAP 值（可能需要几分钟）...')
shap_values = explainer.shap_values(torch.FloatTensor(X_val[:200]).to(device))  # 取 200 个样本

# ========== 6. 可视化 ==========
# 6.1 Summary Plot（全局重要性 + 影响方向）
plt.figure()
shap.summary_plot(
    shap_values, X_val[:200],
    feature_names=all_cols,
    max_display=20,  # 显示 Top 20
    show=False
```

### Permutation Importance vs SHAP 对比

| | Permutation Importance | SHAP |
|---|---|---|
| 计算开销 | 低 | 高（要采样算 Shapley Value） |

## 进阶方法：Partial Dependence Plot（部分依赖图）

### 原理

PDP 回答："当我把某个特征固定在不同值时，模型的平均预测会怎么变？"

$$
PDP(x_j) = \frac{1}{n} \sum_{i=1}^n f(x_j, X_{-j}^{(i)})
$$

对每个样本，把特征 x_j 设成我们想考察的值，其他特征保持原样，取所有样本的平均预测。

### 和特征重要性的区别

- 特征重要性：**"这个特征重不重要？"** → 一个数字
- PDP：**"这个特征到底怎么影响预测？"** → 一条曲线

**举例：在 COVID 数据上跑 public_transit 的 PDP：**

```
预测值
  ↑
  |      ●
  |    ●   ●
  |  ●       ●
  |●           ●
  |               ●
  └──────────────────→ public_transit
  公交使用率越高 → 预测确诊越少（负相关）
```

可以发现：
- 如果曲线是线性的 → 模型学到了简单关系
- 如果曲线有拐点 → 模型学到了非线性关系（可能 overfit）
- 如果曲线震荡 → 特征不重要或 noise

### ICE 图（Individual Conditional Expectation）

PDP 是**平均**效应，ICE 画**每个样本**的效应曲线（细灰线），PDP 是它们的平均（粗黑线）。

ICE 能暴露 PDP 隐藏的问题：如果特征有交互效应，PDP 的"平均"可能掩盖真相。

```python
import numpy as np
import pandas as pd
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset
import matplotlib.pyplot as plt

# ========== 1. 复用数据集和模型 ==========
# （同样复用方法三的 COVID19Dataset 和 NeuralNet）
# 这里展示如何手动实现 PDP，不依赖 sklearn

class COVID19Dataset(Dataset):
    """和方法三一样，略"""
class NeuralNet(nn.Module):
    """和方法三一样，略"""

# ========== 2. 加载模型和数据 ==========
device = 'cuda' if torch.cuda.is_available() else 'cpu'
dataset = COVID19Dataset('covid.train.csv', 'dev', use_all_features=True)
model = NeuralNet(dataset.dim).to(device)
model.load_state_dict(torch.load('models/model.pth', map_location=device, weights_only=True))
model.eval()

all_cols = list(pd.read_csv('covid.train.csv').columns[1:])
feats = list(range(93))

def compute_pdp(feature_name, model, dataset, device, num_grid=50):
    """
    手动计算 Partial Dependence Plot。
    
    原理: 对特征 x_j, 把它固定为 k 个不同的值,
    对每个值, 在所有样本上取平均预测。
    """
    # 找到特征索引
    if feature_name not in all_cols:
        raise ValueError(f'特征 {feature_name} 不存在')
    
    feat_idx = feats[all_cols.index(feature_name)]
    
    # 确定特征的值域（取第 5 到 95 百分位，排除极端值）
    values = dataset.data[:, feat_idx].numpy()
    low, high = np.percentile(values, [5, 95])
    grid = np.linspace(low, high, num_grid)
    
    # 对每个网格点计算平均预测
    pdp_values = []
    orig = dataset.data[:, feat_idx].clone()
    
    with torch.no_grad():
        for val in grid:
            dataset.data[:, feat_idx] = val
            # 预测所有样本
            loader = DataLoader(dataset, batch_size=256)
            preds = []
            for x, _ in loader:
                x = x.to(device)
                preds.append(model(x).cpu().numpy())
            pdp_values.append(np.mean(np.concatenate(preds)))
    
    # 恢复原数据
    dataset.data[:, feat_idx] = orig
    
    return grid, np.array(pdp_values)


def compute_ice(feature_name, model, dataset, device, num_grid=30, sample_n=50):
    """
    手动计算 ICE (Individual Conditional Expectation) 曲线。
    和 PDP 一样但保留每个样本的曲线。
    """
    if feature_name not in all_cols:
        raise ValueError(f'特征 {feature_name} 不存在')
    
    feat_idx = feats[all_cols.index(feature_name)]
    values = dataset.data[:, feat_idx].numpy()
    low, high = np.percentile(values, [5, 95])
    grid = np.linspace(low, high, num_grid)
    
    # 随机选 sample_n 个样本画 ICE
    n = min(sample_n, len(dataset))
    sample_idx = np.random.choice(len(dataset), n, replace=False)
    
    ice_curves = []
    orig = dataset.data[:, feat_idx].clone()
    
    with torch.no_grad():
        for idx in sample_idx:
            sample_curve = []
            sample_x = dataset.data[idx:idx+1].clone()  # shape: (1, dim)
            for val in grid:
                sample_x[0, feat_idx] = val
                pred = model(sample_x.to(device)).cpu().item()
                sample_curve.append(pred)
            ice_curves.append(sample_curve)
    
    dataset.data[:, feat_idx] = orig
    return grid, np.array(ice_curves)


# ========== 3. 画 PDP（全部特征太慢，找几个重要的跑） ==========
important_features = ['tested_positive.1', 'cli.2', 'public_transit', 
                      'worried_finances', 'spent_time']

                             num_grid=20, sample_n=20)
    for i in range(ice.shape[0]):
        ax.plot(grid, ice[i], 'gray', alpha=0.3, linewidth=0.5)
    
    ax.set_xlabel(feat_name)
    ax.set_ylabel('Predicted tested_positive.2')
    ax.set_title(f'PDP + ICE: {feat_name}')
    ax.legend()
    ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('pdp_ice_plots.png', dpi=150)
print('PDP + ICE 图已保存: pdp_ice_plots.png')

def compute_2d_pdp(feat1_name, feat2_name, model, dataset, device, 
                    num_grid=20):
    """两个特征的部分依赖图，看交互效应"""
    f1_idx = feats[all_cols.index(feat1_name)]
    f2_idx = feats[all_cols.index(feat2_name)]
    
    v1 = dataset.data[:, f1_idx].numpy()

    grid1 = np.linspace(np.percentile(v1, 5), np.percentile(v1, 95), num_grid)
    grid2 = np.linspace(np.percentile(v2, 5), np.percentile(v2, 95), num_grid)
    
    pdp_2d = np.zeros((num_grid, num_grid))
    orig1, orig2 = dataset.data[:, f1_idx].clone(), dataset.data[:, f2_idx].clone()

                loader = DataLoader(dataset, batch_size=256)
                preds = []
                for x, _ in loader:
                    x = x.to(device)
                    preds.append(model(x).cpu().numpy())
                pdp_2d[i, j] = np.mean(np.concatenate(preds))
    
    dataset.data[:, f1_idx] = orig1
    dataset.data[:, f2_idx] = orig2
    
    return grid1, grid2, pdp_2d
```

## 三种方法对比总结

| 方法 | 优点 | 缺点 | 什么时候用 |
|------|------|------|-----------|
| 相关性分析 | 快，一行代码 | 只能看线性关系 | 数据探索第一步 |
| 线性模型权重 | 考虑多变量交互 | 还是只能看线性关系；共线性问题 | 想快速知道特征方向（正/负影响） |
| Permutation Importance | 任意模型都能用，最可靠 | 慢（每个特征要重新推理一次）；高相关特征被低估 | 模型已经训练好了，想做最终判断 |
| SHAP | 全局 + 局部解释；理论扎实 | 计算慢；对复杂模型解释成本高 | 写论文、需要可信解释 |
| PDP + ICE | 看到特征如何影响预测 | 只显示两个特征的交互；计算量大 | 理解特征与目标的关系形态 |

## 做特征分析的研究思维

### 三角验证（Triangulation）

不要只看一种方法。**至少用三种方法，结果一致的才可信**。

| 特征 | 相关性排名 | Ridge 权重排名 | PI 排名 | 结论 |
|------|-----------|---------------|--------|------|
| tested_positive.1 | #1 | #1 | #1 | **确定重要** ✅ |
| public_transit | #7 | #15 | #25 | **不确定** ⚠️ |
| wearing_mask | #38 | #28 | #60 | **不重要** ❌ |

如果三种方法结论不一致——这个特征可能和其他特征有复杂的交互关系。值得深入研究。

### 消融实验（Ablation Study）

这是做研究的金标准。不用统计方法，直接：
1. 用所有特征训练 → 记 loss
2. 去掉怀疑不重要的特征 → 重新训练 → 记 loss
3. 如果 loss 没涨 → 可以删掉
4. 如果 loss 涨了 → 留着

消融实验最可信，但最慢（每次要重新训练）。

### 代码（完整可运行）

```python
# feature_analysis_06_ablation.py
# 消融实验 - 完整可运行脚本
import numpy as np
import pandas as pd
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, Dataset

# ========== 1. 复用数据集和模型定义 ==========
# （同样复用方法三的定义）

# ========== 2. 一次训练函数 ==========
def train_and_eval(feature_indices, name="", device='cpu'):
    """
    用指定的特征子集训练模型并返回 dev loss。
    
    参数:
        feature_indices: list[int], 要使用的特征索引列表
        name: 实验名称
    """
    class DynamicCOVIDDataset(Dataset):
        """动态选择特征的数据集"""
        def __init__(self, file_path, mode, feats):
            data = pd.read_csv(file_path).values[:, 1:].astype(float)
            target = data[:, -1]
            data = data[:, feats]
            
            if mode == 'train':
                idx = [i for i in range(len(data)) if i % 10 != 0]
            elif mode == 'dev':
                idx = [i for i in range(len(data)) if i % 10 == 0]
            
            self.data = torch.FloatTensor(data[idx])
            self.target = torch.FloatTensor(target[idx])
            
            # 归一化
            mean = self.data.mean(dim=0, keepdim=True)
            std = self.data.std(dim=0, keepdim=True)
            std[std == 0] = 1
            self.data = (self.data - mean) / std
            self.dim = self.data.shape[1]
        
        def __getitem__(self, idx):
            return self.data[idx], self.target[idx]
        def __len__(self):
            return len(self.data)
    
    # 加载数据
    tr_ds = DynamicCOVIDDataset('covid.train.csv', 'train', feature_indices)
    dv_ds = DynamicCOVIDDataset('covid.train.csv', 'dev', feature_indices)
    tr_loader = DataLoader(tr_ds, batch_size=128, shuffle=True)
    dv_loader = DataLoader(dv_ds, batch_size=128)
    
    # 定义模型（自适应输入维度）
    model = nn.Sequential(
        nn.Linear(tr_ds.dim, 64), nn.ReLU(),
        nn.Linear(64, 64), nn.ReLU(),
        nn.Linear(64, 1)
    ).to(device)
    
    criterion = nn.MSELoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)
    scheduler = torch.optim.lr_scheduler.ReduceLROnPlateau(
        optimizer, patience=10, factor=0.5
    )
    
    # 训练
    best_loss = float('inf')
    patience = 0
    for epoch in range(300):
        model.train()
        for x, y in tr_loader:
            x, y = x.to(device), y.to(device)
            optimizer.zero_grad()
            loss = criterion(model(x), y)
            loss.backward()
            optimizer.step()
        
        model.eval()
        total_loss = 0
        with torch.no_grad():
            for x, y in dv_loader:
                x, y = x.to(device), y.to(device)
                total_loss += criterion(model(x), y).item() * len(x)
        dev_loss = total_loss / len(dv_ds)
        scheduler.step(dev_loss)
        
        if dev_loss < best_loss:
            best_loss = dev_loss
            patience = 0
        else:
            patience += 1
            if patience > 25:
                break
    
    print(f'  [{name:25s}] dim={tr_ds.dim:2d}  dev loss={best_loss:.4f}')
    return best_loss, tr_ds.dim


# ========== 3. 运行消融实验 ==========
device = 'cuda' if torch.cuda.is_available() else 'cpu'
all_cols = list(pd.read_csv('covid.train.csv').columns[1:])
print(f'设备: {device}')
print()

# 定义特征组（基于之前的 PI 结果）
state_indices = list(range(40))  # 40 个州特征
symptom_indices = list(range(40, 93))  # 其他症状特征

# tested_positive 系列（最核心）
tp_indices = [57, 75]  # tested_positive, tested_positive.1
# .2 系列症状特征（和 target 同时期的特征）
symptom_2_indices = list(range(40+18, 93))  # cli.2, ili.2, etc.

# ========== 实验设计 ==========
experiments = [
    ("全部 93 个特征", list(range(93))),
    ("只有目标系列(tp)", tp_indices),
    ("州 + tp", state_indices + tp_indices),
    ("州 + tp + 症状.2", state_indices + tp_indices + symptom_2_indices),
    ("只有症状特征", symptom_indices),
    ("只有州特征", state_indices),
    ("去掉 state", list(range(40, 93))),  # 只用症状
    ("去掉 .2 症状", list(range(40+18)) + list(range(40+18+18, 93))),
]

print('运行消融实验（每个实验会完整训练一次模型）：')
print('=' * 60)
results = []
for name, feats in experiments:
    loss, dim = train_and_eval(feats, name, device)
    results.append((name, dim, loss))

# ========== 4. 结果汇总 ==========
print()
print('=' * 60)
print('消融实验结果汇总')
print('=' * 60)
results.sort(key=lambda r: r[2])  # 按 loss 排序
for name, dim, loss in results:
    baseline = results[0][2]  # 最好的做 baseline
    diff = loss - baseline
    marker = '✅' if diff <= 0.01 else '⚠️' if diff <= 0.1 else '❌'
    print(f'  {marker} {name:25s}  dim={dim:2d}  loss={loss:.4f}  (Δ={diff:+.4f})')
```

### 迭代策略

```
原始 93 个特征 → 相关性分析筛到 50 个
    → 训练模型 → PI 筛到 30 个
    → 消融实验验证 → 最终保留 25-30 个核心特征
```

每一步去掉的特征都可以记录为什么去掉（相关性低、PI 低、消融不掉分），这样以后写实验报告时直接有依据。

## 实践建议

1. **先用相关性分析**筛选出 top 特征，去掉那些和目标没啥关系的
2. **用全部特征训练一个基线模型**
3. **跑 Permutation Importance** 看每个特征的真实贡献
4. **做消融实验（Ablation Study）**：去掉最不重要的几个特征，看模型掉不掉分。不掉就说明这些特征可以去掉
5. **迭代**：保留重要特征，去掉无关特征，再重新训练
