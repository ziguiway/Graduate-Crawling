# 特征分析：如何找出关键特征

以 HW1 COVID-19 预测为例，教你三种找关键特征的方法。

## 为什么需要特征分析？

模型效果不好时，不一定要调网络结构。**找对特征比调模型更重要。**

这个数据有 93 列特征，哪些才是真正起作用的关键特征？

---

## 方法一：相关性分析（最简单）

### 原理

算每个特征和目标值之间的**相关系数**，取绝对值越大，说明这个特征和目标越"步调一致"。

**公式**（Pearson 相关系数）：
$$
r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2}\sqrt{\sum(y_i - \bar{y})^2}}
$$

- r = 1：完全正相关（特征涨，目标也涨）
- r = -1：完全负相关（特征涨，目标跌）
- r = 0：没线性关系

### 代码

```python
corrs = df.corr()[target].sort_values(key=abs, ascending=False)
```

### 局限

相关性只看**线性关系**。神经网络能学到非线性关系，所以相关性低 ≠ 特征不重要。但对于快速筛查，已经够用了。

---

## 方法二：线性模型权重（看模型学到了啥）

### 原理

训练一个线性回归（带 L2 正则化的 Ridge），然后看每个特征的**权重绝对值**。权重越大，模型越依赖这个特征。

和相关性分析的区别：相关性是单变量分析（只看特征和目标的关系），权重是**多变量分析**（看所有特征一起时各个特征有多大贡献）。可能有特征的权重和相关性符号不同——那就是和其他特征相互作用的结果。

## 方法三：Permutation Importance（置换重要性，最可靠）

### 原理

1. 先训练好模型，算 baseline loss
2. **把某一个特征的值打乱**（破坏它和目标的关联）
3. 再算 loss，看上升多少
4. 上升越多 → 这个特征越重要

**打个比方：** 你想知道方向盘对开车重不重要，就把方向盘拆了开一段路，看还能不能正常开。

### 为什么它最好？

相关性和权重都只看**线性关系**。Permutation Importance 不管你的模型是线性、神经网络还是随机森林，**统统都能用**，直接反映"没这个特征模型会差多少"。

### HW1 的结果

用完整 93 个特征、3 层神经网络训练后：

```

  tested_positive.1         ↑32.53  ████████████████

  tested_positive           ↑1.87   ██████
```

### 关键发现

1. **tested_positive.1 是最重要的特征**——疫情数据前后天高度自相关，昨天啥样今天大概率也啥样。去掉它模型直接崩。

2. **州信息比症状信息更关键**——不同州疫情曲线完全不同（纽约 vs 阿拉巴马），州特征 tells 模型"你在哪个州"，这比其他症状调查数据更关键。

3. **相同特征的不同 lag 重要程度不同**：
   - `.2`（同时间）的症状特征重要：cli.2, ili.2
   - `.1` 和当前的症状重要性较低：模型已经有了 lag-2 的信息就不太需要 lag-1 了

4. **你之前的 feature selection 没选错核心**——`target_only=True` 选了 tested_positive + tested_positive.1 + 40 个州，确实抓住了最重要的特征。但缺点是漏了 cli.2、ili.2、hh_cmnty_cli.2 等次重要特征，这些加起来能把 loss 从 0.98 降到 0.88。

### 代码

```python
def permutation_importance(model, dv_loader, dv_dataset, criterion, device='cpu'):
    model.eval()
    
    # baseline loss
    total_loss = 0
    for x, y in dv_loader:
        x, y = x.to(device), y.to(device)
        with torch.no_grad():
            total_loss += criterion(model(x), y).item() * len(x)
    baseline = total_loss / len(dv_dataset)
    
    importances = []
    for feat_idx in range(dv_dataset.dim):
        orig = dv_dataset.data[:, feat_idx].clone()
        dv_dataset.data[:, feat_idx] = orig[torch.randperm(len(dv_dataset))]
        
        total_loss = 0
        for x, y in dv_loader:
            x, y = x.to(device), y.to(device)
            with torch.no_grad():
                total_loss += criterion(model(x), y).item() * len(x)
        
        dv_dataset.data[:, feat_idx] = orig  # 还原
        importances.append((feat_names[feat_idx], total_loss/len(dv_dataset) - baseline))
    
    return sorted(importances, key=lambda x: x[1], reverse=True)
```

## 三种方法对比总结

| 方法 | 优点 | 缺点 | 什么时候用 |
|------|------|------|-----------|
| 相关性分析 | 快，一行代码 | 只能看线性关系 | 数据探索第一步 |
| 线性模型权重 | 考虑多变量交互 | 还是只能看线性关系 | 想快速知道特征方向（正/负影响） |
| Permutation Importance | 任意模型都能用，最可靠 | 慢（每个特征要重新推理一次） | 模型已经训练好了，想做最终判断 |

---

## 实践建议

1. **先用相关性分析**筛选出 top 特征，去掉那些和目标没啥关系的
2. **用全部特征训练一个基线模型**
3. **跑 Permutation Importance** 看每个特征的真实贡献
4. **做消融实验（Ablation Study）**：去掉最不重要的几个特征，看模型掉不掉分。不掉就说明这些特征可以去掉
5. **迭代**：保留重要特征，去掉无关特征，再重新训练

## 对你 HW1 的直接建议

从上面的分析来看，只用 `target_only=True` 那 42 个特征是不够的。建议用**全部 93 个特征**，并且对所有特征做归一化。这样 dev loss 可以从 0.98 降到 0.88 左右，离排行榜的 0.8 就更近了。
