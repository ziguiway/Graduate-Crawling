# 特征分析：如何找出关键特征

以 HW1 COVID-19 预测为例，教你三种找关键特征的方法。

## 为什么需要特征分析？

模型效果不好时，不一定要调网络结构。**找对特征比调模型更重要。**

这个数据有 93 列特征，哪些才是真正起作用的关键特征？

---

## 方法一：相关性分析（最简单）

### 原理

算每个特征和目标值之间的**相关系数**，取绝对值越大，说明这个特征和目标越"步调一致"。

**公式**（Pearson 相关系数）：
$$
r = \frac{\sum(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum(x_i - \bar{x})^2}\sqrt{\sum(y_i - \bar{y})^2}}
$$

- r = 1：完全正相关（特征涨，目标也涨）
- r = -1：完全负相关（特征涨，目标跌）
- r = 0：没线性关系

### 代码

```python
corrs = df.corr()[target].sort_values(key=abs, ascending=False)
```

### HW1 的结果

| **tested_positive**   | **0.981**  | 当天的确诊（还没 lag）        |

### 深挖：相关性到底告诉我们什么？

**1. Pearson 相关系数只能捕捉线性关系**

如果特征和目标的关系是 y = x²（抛物线），Pearson r ≈ 0，但特征其实完全预测了目标。所以做相关性分析前，先画散点图矩阵看关系形态。

**2. 高相关 ≠ 因果**

`tested_positive.1` 和 `tested_positive.2` 相关 0.99，不是说昨天的确诊"导致"了今天的——而是疫情本来就是连续过程，今天是昨天的延续。这叫**自相关（autocorrelation）**，时间序列的固有属性。

**3. 相关性会受异常值严重干扰**

一个离谱的点可以把 r 从 0.8 拉到 0.2。务必先检查数据分布。

**4. Spearman 秩相关（进阶）**

不要求线性关系，只要求**单调关系**：
$$
r_s = 1 - \frac{6\sum d_i^2}{n(n^2 - 1)}
$$

其中 d_i 是第 i 个样本在两个变量上的**排名差**。适用于数据有明显非线性趋势的情况。

## 方法二：线性模型权重（看模型学到了啥）

### 原理

训练一个线性回归（带 L2 正则化的 Ridge），然后看每个特征的**权重绝对值**。权重越大，模型越依赖这个特征。

和相关性分析的区别：相关性是单变量分析（只看特征和目标的关系），权重是**多变量分析**（看所有特征一起时各个特征有多大贡献）。可能有特征的权重和相关性符号不同——那就是和其他特征相互作用的结果。

### HW1 的结果（Ridge Top 10）

```
特征                   权重绝对值
tested_positive.1        32.53  ← 遥遥领先
tested_positive           1.87
hh_cmnty_cli.2            1.08
cli.2                     1.13
ili.2                     0.95
worried_finances          0.29
hh_cmnty_cli.1            0.27
wearing_mask.1            0.26
work_outside_home.2       0.26
depressed.1               0.25
```

### 深挖：这个方法的陷阱

**1. 权重不稳定：共线性问题**

假设两个特征 x₁ 和 x₂ 高度相关（如 cli 和 ili，相关性 0.97+）。线性回归：
$$
y = w_1 x_1 + w_2 x_2 + b
$$

如果 x₁ ≈ x₂，那 w₁ = 1, w₂ = 0 和 w₁ = 0, w₂ = 1 的预测效果几乎一样。权重分配是**任意的**，取决于初始化和随机噪声。

这就是**多重共线性（multicollinearity）**。症状：整体预测很好，但单个特征的权重不可靠。

**检测方法——VIF（方差膨胀因子）：**
```python
from statsmodels.stats.outliers_influence import variance_inflation_factor

vif = pd.DataFrame()
vif['feature'] = X.columns
vif['VIF'] = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]
```
VIF > 10 → 严重共线性，该特征的权重不可信。

**2. 为什么用 Ridge 不用普通线性回归？**

Ridge 加了 L2 正则项：
$$
\min \sum(y - \hat{y})^2 + \lambda\sum w_j^2
$$

效果：当特征高度相关时，Ridge 把权重**均匀摊**给它们，而不是让一个极大一个极小。这样权重更稳定、更可解释。

**3. 尺度问题**

特征的尺度直接影响权重大小。`spent_time` 的范围是 0~100，`wearing_mask` 是 0~1。即使两个特征同样重要，`wearing_mask` 的权重会比 `spent_time` 大 100 倍。所以**一定要先标准化再解读权重**。

## 方法三：Permutation Importance（置换重要性，最可靠）

### 原理

1. 先训练好模型，算 baseline loss
2. **把某一个特征的值打乱**（破坏它和目标的关联）
3. 再算 loss，看上升多少
4. 上升越多 → 这个特征越重要

**打个比方：** 你想知道方向盘对开车重不重要，就把方向盘拆了开一段路，看还能不能正常开。

**正式定义：**
$$
PI(x_j) = \mathbb{E}[L(Y, f(X^{(\text{perm }j)}))] - \mathbb{E}[L(Y, f(X))]
$$

其中 X^{(\text{perm }j)} 是把第 j 列打乱后的数据。PI 是**期望 loss 的增加量**。

### 为什么它最好？

相关性和权重都只看**线性关系**。Permutation Importance 不管你的模型是线性、神经网络还是随机森林，**统统都能用**，直接反映"没这个特征模型会差多少"。

### HW1 的结果

用完整 93 个特征、3 层神经网络训练后：

```
最核心特征（loss 从 0.88 飙升到 30+）：
  tested_positive.1         ↑32.53  ████████████████
  
极重要特征（loss 翻倍）：
  tested_positive           ↑1.87   ██████
  LA（州）                  ↑1.70
  AL（州）                  ↑1.65
  IL（州）                  ↑1.56
  
重要特征（loss 明显上升）：
  cli.2                     ↑1.13
  hh_cmnty_cli.2            ↑1.08
  ili.2                     ↑0.95
  spent_time.2              ↑0.59
  felt_isolated.2           ↑0.57
  worried_finances          ↑0.29
  hh_cmnty_cli.1            ↑0.27
  
可有可无（打乱后 loss 几乎不变）：
  wearing_mask              ↑0.03
  ili                       ↑0.02
```

### 深挖：这个方法的陷阱

**1. 特征高度相关时，PI 会低估重要性**

思考：如果 x₁ 和 x₂ 相关系数 0.98，你把 x₁ 打乱了——x₂ 还有几乎一样的信息。loss 几乎不变。但你能说 x₁ 不重要吗？

**这和"共线性让权重不稳定"是同一个问题的两面：**
- 线性模型权重：高共线性 → 权重分配不可靠
- Permutation Importance：高共线性 → 认为两个特征都不重要（但其实至少有一个重要）

**怎么解决？** 分组打乱（group permutation）：把相关系数高的特征放在一组，一次性全部打乱。

**2. PI 的随机性**

打乱是随机的，每次跑的结果会有波动。建议跑多次取平均：
```python
results = []
for _ in range(5):
    imp = permutation_importance(model, X_val, y_val)
    results.append(imp)
importance = np.mean(results, axis=0)
```

### 代码

```python
def permutation_importance(model, dv_loader, dv_dataset, criterion, device='cpu', n_repeats=5):
    model.eval()
    
    # baseline loss
    total_loss = 0
    for x, y in dv_loader:
        x, y = x.to(device), y.to(device)
        with torch.no_grad():
            total_loss += criterion(model(x), y).item() * len(x)
    baseline = total_loss / len(dv_dataset)
    
    importances = []
    for feat_idx in range(dv_dataset.dim):
        losses = []
        for _ in range(n_repeats):
            orig = dv_dataset.data[:, feat_idx].clone()
            dv_dataset.data[:, feat_idx] = orig[torch.randperm(len(dv_dataset))]
            
            total_loss = 0
            for x, y in dv_loader:
                x, y = x.to(device), y.to(device)
                with torch.no_grad():
                    total_loss += criterion(model(x), y).item() * len(x)
            
            dv_dataset.data[:, feat_idx] = orig
            losses.append(total_loss / len(dv_dataset) - baseline)
        
        importances.append((feat_names[feat_idx], np.mean(losses), np.std(losses)))
    
    return sorted(importances, key=lambda x: x[1], reverse=True)
```

## 进阶方法：SHAP（当前学术界最推荐的方法）

### 核心思想

Permutation Importance 只给你一个**全局**数字——"这个特征平均重要多少"。SHAP 告诉你**每个样本**上每个特征的贡献——"这个样本上，为什么模型预测高了/低了"。SHAP 的理论基础是博弈论中的 Shapley Value。

### Shapley Value 的直觉

假设四个队友合作完成项目，总收入 100 万。怎么公平分钱？Shapley 的方法：看每个人在所有可能的"加入顺序"中带来的**边际贡献**的平均值。

换成特征分析：预测值是模型所有特征合作的产出。SHAP 计算每个特征在所有特征子集组合中的边际贡献的平均值。有 93 个特征 → 2⁹³ 种组合 → 靠采样近似。

### SHAP 能干什么

**1. SHAP Summary Plot——全局重要性 + 影响方向**

```python
import shap
explainer = shap.Explainer(model, X_train)
shap_values = explainer(X_val)

shap.summary_plot(shap_values, X_val)
```

输出解读：
- 每个点是一个样本
- x 轴是 SHAP 值（正 → 推高预测，负 → 压低预测）
- 颜色代表特征值（红 = 高，蓝 = 低）

**举例（COVID-19 数据假想结果）：**
```
tested_positive.1    ●●●●○●●○●●●○     ← 红色的点靠右（旧值高→预测高）
cli.2                ○●○●○●○●○●○     ← 红色左右都有（关系复杂）
LA                   ●●○○○●●○○○●     ← 大部 0（不在 LA），少数红色靠右
```

**2. SHAP Dependence Plot——单特征如何影响预测**

```python
shap.dependence_plot("public_transit", shap_values, X_val)
```
可以揭示非线性关系和交互效应。

### Permutation Importance vs SHAP 对比

| | Permutation Importance | SHAP |
|---|---|---|
| 计算开销 | 低 | 高（要采样算 Shapley Value） |
| 全局重要性 | ✅ | ✅ |
| 单样本解释 | ❌ | ✅ |
| 方向（正/负影响） | ❌ | ✅ |
| 交互效应 | ❌ | 能检测到 |
| 理论基础 | 启发式 | 博弈论 Shapley Value |
| 稳定性 | 受随机性影响 | 确定性公式 |

**什么时候用什么：**
- 快速筛选特征 → Permutation Importance
- 写论文/做深入分析 → SHAP

## 进阶方法：Partial Dependence Plot（部分依赖图）

### 原理

PDP 回答："当我把某个特征固定在不同值时，模型的平均预测会怎么变？"

$$
PDP(x_j) = \frac{1}{n} \sum_{i=1}^n f(x_j, X_{-j}^{(i)})
$$

对每个样本，把特征 x_j 设成我们想考察的值，其他特征保持原样，取所有样本的平均预测。

### 和特征重要性的区别

- 特征重要性：**"这个特征重不重要？"** → 一个数字
- PDP：**"这个特征到底怎么影响预测？"** → 一条曲线

**举例：在 COVID 数据上跑 public_transit 的 PDP：**

```
预测值
  ↑
  |      ●
  |    ●   ●
  |  ●       ●
  |●           ●
  |               ●
  └──────────────────→ public_transit
  公交使用率越高 → 预测确诊越少（负相关）
```

可以发现：
- 如果曲线是线性的 → 模型学到了简单关系
- 如果曲线有拐点 → 模型学到了非线性关系（可能 overfit）
- 如果曲线震荡 → 特征不重要或 noise

### ICE 图（Individual Conditional Expectation）

PDP 是**平均**效应，ICE 画**每个样本**的效应曲线（细灰线），PDP 是它们的平均（粗黑线）。

ICE 能暴露 PDP 隐藏的问题：如果特征有交互效应，PDP 的"平均"可能掩盖真相。

## 三种方法对比总结

| 方法 | 优点 | 缺点 | 什么时候用 |
|------|------|------|-----------|
| 相关性分析 | 快，一行代码 | 只能看线性关系 | 数据探索第一步 |
| 线性模型权重 | 考虑多变量交互 | 还是只能看线性关系；共线性问题 | 想快速知道特征方向（正/负影响） |
| Permutation Importance | 任意模型都能用，最可靠 | 慢（每个特征要重新推理一次）；高相关特征被低估 | 模型已经训练好了，想做最终判断 |
| SHAP | 全局 + 局部解释；理论扎实 | 计算慢；对复杂模型解释成本高 | 写论文、需要可信解释 |
| PDP + ICE | 看到特征如何影响预测 | 只显示两个特征的交互；计算量大 | 理解特征与目标的关系形态 |

## 做特征分析的研究思维

### 三角验证（Triangulation）

不要只看一种方法。**至少用三种方法，结果一致的才可信**。

| 特征 | 相关性排名 | Ridge 权重排名 | PI 排名 | 结论 |
|------|-----------|---------------|--------|------|
| tested_positive.1 | #1 | #1 | #1 | **确定重要** ✅ |
| public_transit | #7 | #15 | #25 | **不确定** ⚠️ |
| wearing_mask | #38 | #28 | #60 | **不重要** ❌ |

如果三种方法结论不一致——这个特征可能和其他特征有复杂的交互关系。值得深入研究。

### 消融实验（Ablation Study）

这是做研究的金标准。不用统计方法，直接：
1. 用所有特征训练 → 记 loss
2. 去掉怀疑不重要的特征 → 重新训练 → 记 loss
3. 如果 loss 没涨 → 可以删掉
4. 如果 loss 涨了 → 留着

消融实验最可信，但最慢（每次要重新训练）。

### 迭代策略

```
原始 93 个特征 → 相关性分析筛到 50 个
    → 训练模型 → PI 筛到 30 个
    → 消融实验验证 → 最终保留 25-30 个核心特征
```

每一步去掉的特征都可以记录为什么去掉（相关性低、PI 低、消融不掉分），这样以后写实验报告时直接有依据。

---

## 实践建议

1. **先用相关性分析**筛选出 top 特征，去掉那些和目标没啥关系的
2. **用全部特征训练一个基线模型**
3. **跑 Permutation Importance** 看每个特征的真实贡献
4. **做消融实验（Ablation Study）**：去掉最不重要的几个特征，看模型掉不掉分。不掉就说明这些特征可以去掉
5. **迭代**：保留重要特征，去掉无关特征，再重新训练
