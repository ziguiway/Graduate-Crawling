# 科研入门指南

## 一、科研全流程


```
研究方向 → 具体问题 → 技术创新 → 实验评估 → 论文写作
```

## 二、选题

**好方向的四个条件：** 有实际需求、新颖、有未知空间、实验室有积累

**两类问题：**

|     | 新问题          | 老问题            |
| --- | ------------ | -------------- |
| 目标  | 定义并解决没人做过的问题 | 用更好的方法解决已知问题   |
| 贡献  | 问题定义 + 技术方案  | 技术创新（更快/更高/更强） |
| 关键  | 有真实背景和数据     | 在Benchmark上打榜  |

## 三、读论文

- **找方向**：广度优先，看idea不看细节
- **做研究**：深度优先，看细节、复现代码、批判式阅读
- **90%精力**读顶会顶刊，**10%**读交叉领域
- 工具：谷歌学术、Connected Papers、DBLP

## 四、做研究的素质

**要做到：** 脚踏实地、快速迭代、多总结、保证科研时间

**大忌：** 没兴趣、不了解前人工作、浮躁急于求成、马虎、懒惰

**师生关系：** 目标一致、相互欣赏、共同进步、契约精神

## 五、写论文

**论文结构：** 摘要 → 引言 → 相关工作 → 框架 → 技术细节(2~3个创新点) → 实验 → 总结

**摘要四要素：** What(什么问题) → Why(为什么写) → How(怎么做) → So What(效果如何)

**核心原则：**
- ⚠️ **初稿严禁用ChatGPT生成**，可用来检查语法
- **结构比英语更重要**，逻辑清晰、图文并茂
- 引言几乎决定能否录取，要把故事讲好
- 把读者当"傻子"，循序渐进，用例子解释复杂内容
- 相关工作多夸别人（审稿人可能就是作者）
- 画图要用心，一个图想两天画一天很正常

**审稿人想看到：** 新问题 / 新方法 / 好故事 / 好排版

**审稿人讨厌：** 老问题简单拼凑 / 差的写作和图 / 实验设置不当

## 六、投稿心态

- 顶会接受率仅~20%，被拒是常态
- 找原因、补短板、根据意见修改
- 伤心一两天就过去

## 💡 个人感悟

看完这篇辅导，我终于知道科研是怎么回事了——**原来就是一场高级版的"找茬游戏"**：

1. **先找方向的茬**：读一堆顶会论文，跟导师师兄师姐套近乎，看看哪个坑还没人跳
2. **再找问题的茬**：新问题（没人做过）or 老问题（做得不够好），反正总得挑点毛病
3. **搞技术创新**：憋出2-3个创新点，像挤牙膏一样痛苦但必须有
4. **做实验评估**：跟Baseline各种PK，证明我的方法"更快更高更强"
5. **写论文**：把上面的故事讲圆了，让审稿人觉得"这小伙子/姑娘有点东西"

**老师提到的那些工具和网站**（谷歌学术、Connected Papers、DBLP...）已经默默躺进我的收藏夹吃灰了，后续一定去逛（flag先立为敬）。

现在对科研有了大概了解，感觉就像：**知道了游戏规则，但还没开始打怪升级**。不过至少不会像个无头苍蝇一样乱撞了。

> *"三四月做的事情，八九月自有答案"* —— 希望到时候我的答案不是"这题我不会" 😂
