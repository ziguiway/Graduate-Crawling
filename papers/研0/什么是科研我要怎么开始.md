# 科研入门指南

## 一、科研全流程


```
研究方向 → 具体问题 → 技术创新 → 实验评估 → 论文写作
```

## 二、选题

**好方向的四个条件：** 有实际需求、新颖、有未知空间、实验室有积累

**两类问题：**

|     | 新问题          | 老问题            |
| --- | ------------ | -------------- |
| 目标  | 定义并解决没人做过的问题 | 用更好的方法解决已知问题   |
| 贡献  | 问题定义 + 技术方案  | 技术创新（更快/更高/更强） |
| 关键  | 有真实背景和数据     | 在Benchmark上打榜  |

## 三、读论文

- **找方向**：广度优先，看idea不看细节
- **做研究**：深度优先，看细节、复现代码、批判式阅读
- **90%精力**读顶会顶刊，**10%**读交叉领域
- 工具：谷歌学术、Connected Papers、DBLP

## 四、做研究的素质

**要做到：** 脚踏实地、快速迭代、多总结、保证科研时间

**大忌：** 没兴趣、不了解前人工作、浮躁急于求成、马虎、懒惰

**师生关系：** 目标一致、相互欣赏、共同进步、契约精神

## 五、写论文

**论文结构：** 摘要 → 引言 → 相关工作 → 框架 → 技术细节(2~3个创新点) → 实验 → 总结

**摘要四要素：** What(什么问题) → Why(为什么写) → How(怎么做) → So What(效果如何)

**核心原则：**
- ⚠️ **初稿严禁用ChatGPT生成**，可用来检查语法
- **结构比英语更重要**，逻辑清晰、图文并茂
- 引言几乎决定能否录取，要把故事讲好
- 把读者当"傻子"，循序渐进，用例子解释复杂内容
- 相关工作多夸别人（审稿人可能就是作者）
- 画图要用心，一个图想两天画一天很正常

**审稿人想看到：** 新问题 / 新方法 / 好故事 / 好排版

**审稿人讨厌：** 老问题简单拼凑 / 差的写作和图 / 实验设置不当

## 六、投稿心态

- 顶会接受率仅~20%，被拒是常态
- 找原因、补短板、根据意见修改
- 伤心一两天就过去

## 个人感悟

看完这篇辅导，大概懂了科研是啥回事——**说白了就是找茬+讲故事**：

先读一堆论文找方向，然后挑个问题（新的旧的都行），想办法做得比别人好，跑实验验证，最后写成论文忽悠审稿人。

老师提到的那些工具网站，谷歌学术、Connected Papers、DBLP之类的，收藏了，后面慢慢研究。现在就是知道个大概流程，具体咋操作还得边做边学。

反正记住几个关键词：**别浮躁、多交流、保证科研时间、被拒很正常**。
